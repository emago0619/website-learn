# ランディングページ要件定義

## プロジェクトの概要
- **目的**: 新サービスやキャンペーンを紹介し、資料請求・お問い合わせ・購入などのアクションにつなげる。
- **主な利用者**: プロダクトを短期間で告知したいマーケターや事業担当者。
- **完成イメージ**: ファーストビューで価値を訴求し、中盤で機能や導入実績を紹介。最後はFAQとCTAで締める1ページ構成。

## 必須要件
1. ヒーローセクション（キャッチコピー、サブコピー、メインCTAボタン）
2. 特徴／メリット紹介（3〜4項目、アイコン付き）
3. 顧客の声（写真＋コメントを3件以上掲載）
4. 料金プラン比較表（ライト／スタンダード／プレミアムなど）
5. FAQ（5項目以上、アコーディオン表示）
6. スクロールに合わせたアニメーション（フェードインなど）

## 任意要件（余力があれば）
- CTAボタンのA/Bテスト用バリエーション
- カウントダウンタイマー（期間限定キャンペーン向け）
- ヒートマップ計測スクリプトの埋め込み

## UI/UXのポイント
- ファーストビューに背景画像やグラデーションを活用して印象づける
- CTAボタンは常に視認性の高い色を使用し、スクロール後も追従ボタンを表示する
- レスポンシブ対応で縦長スマホでも読みやすいレイアウトを保つ

## 技術スタック
- HTML
- CSS
- 軽量なJavaScript（アニメーションはIntersection Observerなど）

## 成果物チェックリスト
- `index.html`（単一ページ構成）
- 画像アセット一式
- `styles.css`
- `scripts.js`
- CTA運用方法をまとめたREADME
