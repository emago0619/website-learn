# ブログサイト要件定義

## プロジェクトの概要
- **目的**: Markdownベースで記事を管理し、個人がスムーズに情報発信できるブログを構築する。
- **主な利用者**: 技術ブログを始めたい初学者、情報発信を習慣化したい個人。
- **完成イメージ**: トップページに記事一覧、詳細ページはMarkdownから生成し、カテゴリやタグで整理できる仕組み。

## 必須要件
1. 記事一覧ページ（最新順表示、カテゴリやタグで絞り込み）
2. 記事詳細ページ（MarkdownをHTMLに変換し、目次を自動生成）
3. 検索機能（タイトル・本文の全文検索）
4. RSSフィード生成（購読用）
5. コードブロックのシンタックスハイライト
6. SEO対策（メタタグ、OGP設定）

## 任意要件（余力があれば）
- コメント機能（GitHub IssuesやDisqusなどの外部サービス連携）
- 記事の関連記事表示
- サイトマップXMLの自動生成

## UI/UXのポイント
- トップページでカテゴリごとに色分けしたバッジを表示
- 長文でも読みやすい余白と行間を確保
- 検索結果がすぐ分かるようリアルタイム検索または絞り込み結果の件数表示を行う

## 技術スタック
- 静的サイトジェネレーター（Hugo、Astro、11tyなど）またはPure HTML＋静的ビルドスクリプト

## 成果物チェックリスト
- 記事用Markdownファイルをまとめたフォルダ
- ビルドスクリプト一式
- 公開用の`public/`フォルダ
- デプロイ手順を記したREADME
