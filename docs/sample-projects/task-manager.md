# タスク管理アプリ要件定義（Next.js + Supabase）

## プロジェクトの概要
- **目的**: 認証とデータベース連携を含むフルスタックなタスク管理ツールを作り、チーム利用を想定した設計を学ぶ。
- **主な利用者**: タスクを共有したい小規模チーム、フルスタック開発の練習をしたい学習者。
- **完成イメージ**: Kanban風ボードとリスト表示を切り替えられるダッシュボード型Webアプリ。

## 必須要件
1. Supabaseを利用したユーザー認証（メール＋OAuth）
2. タスクのCRUD（タイトル、詳細、期限、優先度、担当者、タグ）
3. カンバンボード（ドラッグ＆ドロップでステータス変更）
4. チーム共有（組織単位でのタスク閲覧、招待リンク）
5. リアルタイム更新（チームメンバーの操作を即時反映）
6. アクティビティログ（タスク変更履歴を表示）

## 任意要件（余力があれば）
- 通知機能（期限前リマインドメール、アプリ内通知）
- カレンダー表示（月間／週間ビュー）
- 外部連携（Google CalendarやSlack Webhook）

## UI/UXのポイント
- サイドバー＋メインコンテンツのダッシュボードレイアウト
- モーダルでタスク詳細を確認・編集できる導線
- カラーラベルやステータスバッジで状態を直感的に把握できる工夫
- モバイル閲覧時はリスト表示に切り替えるレスポンシブ設計

## 技術スタック
- Next.js 14
- TypeScript
- Supabase
- Tailwind CSS
- ドラッグ＆ドロップ: `dnd-kit` または `@hello-pangea/dnd`
- 状態管理: `zustand` などの軽量ライブラリ
