静的サイトプロジェクト

ポートフォリオサイト

目的個人のスキルや実績を整理し、採用担当者やクライアントに魅力を伝える。
想定利用者就職・転職活動中のエンジニア／デザイナー、フリーランスのクリエイター。
完成イメージシングルページの静的サイト。ヒーロー、スキル、実績、連絡先の4つの大きなセクションで構成。
技術スタックHTML、CSS（またはTailwind CSSなどのユーティリティ系CSS）、軽いJavaScript。
成果物index.html、/assets配下の画像、styles.css、READMEに環境構築・カスタマイズ手順を記載。

必須機能

  • 自己紹介セクション（プロフィール、肩書き、SNSリンク）
  • スキル一覧（カテゴリーごとにアイコンやバッジ表示）
  • 代表的な制作実績3件以上（画像＋説明＋詳細ページへの導線）
  • 連絡手段（メールフォーム、またはメールアドレスの記載）
  • レスポンシブ対応（スマホ幅での崩れ防止）
  • ダークモード切り替えスイッチ

任意機能（余裕があれば）

  • ブログ記事やニュースの最新情報
  • PDF形式の職務経歴書ダウンロード
  • Google Analyticsによるアクセス解析

UI/UX要件

  • 色は落ち着いたメインカラー＋アクセントカラーの2色構成
  • セクション間の余白を十分に取り、スクロールで読みやすく
  • ダークモード時は背景色・文字色が適切なコントラスト比になるよう調整

ブログサイト

目的Markdownベースで記事を管理できる小規模な個人ブログを構築し、情報発信をスムーズにする。
想定利用者技術ブログを始めたい初学者、情報発信を習慣化したい個人。
完成イメージトップページに記事一覧、記事詳細はMarkdownから生成、カテゴリ・タグで整理できる仕組み。
技術スタック静的サイトジェネレーター（Hugo、Astro、11tyなど）またはPure HTML＋静的ビルドスクリプト。
成果物記事用Markdownフォルダ、ビルドスクリプト、public/フォルダ、デプロイ手順付きREADME。

必須機能

  • 記事一覧ページ（最新順、カテゴリやタグで絞り込み）
  • 記事詳細ページ（MarkdownをHTMLに変換、目次自動生成）
  • 検索機能（タイトル・本文からの全文検索）
  • RSSフィード生成（購読用）
  • コードブロックのシンタックスハイライト
  • SEO対策（メタタグ、OGP設定）

任意機能（余裕があれば）

  • コメント機能（GitHub Issues、Disqusなど外部サービス連携）
  • 記事の関連記事表示
  • サイトマップXMLの自動生成

UI/UX要件

  • トップページでカテゴリ別の色分けバッジを使用
  • 長文記事でも読みやすい余白と行間を確保
  • 検索結果が即座に分かるよう、リアルタイム検索 or 絞り込み後の件数表示

ランディングページ

目的新サービスやキャンペーンを紹介し、資料請求・お問い合わせ・購入ボタンなどの行動を促す。
想定利用者プロダクトを短期間で告知したいマーケターや事業担当者。
完成イメージファーストビューでサービス価値を伝え、中盤で機能・導入実績、最後にFAQとCTAで締める1ページ構成。
技術スタックHTML、CSS、軽量JavaScript（アニメーション用にIntersection Observerなど）
成果物index.html単一ページ、画像アセット、styles.css、scripts.js、CTAの運用方法を記したREADME。

必須機能

  • ヒーローセクション（キャッチコピー、サブコピー、メインCTAボタン）
  • 特徴／メリット紹介（3～4項目、アイコン付き）
  • 顧客の声（写真＋コメント 3件以上）
  • 料金プラン比較表（ライト／スタンダード／プレミアムなど）
  • FAQ（5項目以上、アコーディオン表示）
  • スクロールに合わせたアニメーション（フェードイン等）

任意機能（余裕があれば）

  • CTAボタンのA/Bテスト用バリエーション
  • カウントダウンタイマー（期間限定キャンペーン向け）
  • ヒートマップ計測スクリプト埋め込み

UI/UX要件

  • ファーストビューの背景画像またはグラデーションを活用
  • CTAボタンは常に視認性の高い色を使用し、スクロール後も追従ボタンを表示
  • レスポンシブ時のレイアウト崩れを防ぐ（縦長スマホでの可読性重視）

動的サイトプロジェクト（React/Next.js）

コーポレートサイト

目的Next.jsを使い、更新しやすく表示速度の速い企業サイトを構築する。
想定利用者企業のWeb担当者、Next.jsの学習者。
完成イメージトップ／サービス／お知らせ／会社情報／採用／お問い合わせを備えた多ページサイト。
技術スタックNext.js 14、TypeScript、Tailwind CSS、next-seo、フォーム送信用にreact-hook-form＋@tanstack/react-query。
成果物Next.jsプロジェクト一式、CMS連携用の設定例（例: Contentfulのモデル定義JSON）、READMEに環境変数とデプロイ手順を明記。

必須機能

  • 静的生成（SSG）またはISRによる高速表示
  • お知らせ一覧・詳細ページ（ヘッドレスCMSと連携できる設計）
  • 採用情報一覧（職種別、勤務地、雇用形態のタグ表示）
  • お問い合わせフォーム（送信後のサンクスページ／メール通知）
  • 多言語対応（日本語・英語切替、next-intlなどを利用）
  • アクセシビリティを意識したナビゲーション（キーボード操作・ARIA対応）

任意機能（余裕があれば）

  • サイト内検索（AlgoliaなどのSaaS連携）
  • 事例紹介ページ（CMSの別コンテンツタイプ）
  • サイトパフォーマンス計測（Lighthouseレポート、Core Web Vitals計測）

UI/UX要件

  • ブランドカラーをもとにしたデザインシステム（ボタン、カード、見出しなどのコンポーネント定義）
  • PC/スマホで共通ヘッダー、フッター、パンくずリストを表示
  • ナビゲーション固定ヘッダー＋スクロール時の影付与

タスク管理アプリ

目的認証・データベース連携を含むフルスタックなタスク管理ツールを作り、チーム利用を想定した設計を学ぶ。
想定利用者チームでタスク共有したい小規模事業者、フルスタック開発を練習したい学習者。
完成イメージKanban風ボードとリスト表示を切り替えられるダッシュボード型Webアプリ。
技術スタックNext.js 14、TypeScript、Supabase、Tailwind CSS、dnd-kitまたは@hello-pangea/dnd、状態管理はzustandなど軽量なもの。
成果物Next.jsプロジェクト、Supabaseのテーブル定義SQL、.envテンプレート、主要コンポーネントのStorybook（任意）、READMEにセットアップ手順と運用注意点を記載。

必須機能

  • Supabaseを利用したユーザー認証（メール＋OAuth）
  • タスクのCRUD（タイトル、詳細、期限、優先度、担当者、タグ）
  • カンバンボード（ドラッグ＆ドロップでステータス変更）
  • チーム共有（組織単位でのタスク閲覧、招待リンク）
  • リアルタイム更新（チームメンバーの操作を即時反映）
  • アクティビティログ（タスクの変更履歴を表示）

任意機能（余裕があれば）

  • 通知機能（期限前リマインドメール、アプリ内通知）
  • カレンダー表示（月間／週間ビュー）
  • 外部連携（Google Calendar、Slack Webhook）

UI/UX要件

  • ダッシュボードレイアウト（サイドバー＋メインコンテンツ）
  • モーダルでタスク詳細を確認・編集できる
  • カラーラベルやステータスバッジで状態が直感的に分かる
  • モバイル時はリスト表示に切り替えるレスポンシブ設計

このドキュメントの使い方

  1. 作りたいプロジェクトを選び、要件を上から順に満たす形で設計・実装する。
  2. 「必須機能」を完了ラインとして定義し、公開や社内共有ができる状態を目指す。
  3. 「任意機能」は余裕があるときに追加し、発展課題として活用する。
  4. READMEやNotionなどの管理ツールにこのページのリンクを貼り、進捗管理に役立てる。
