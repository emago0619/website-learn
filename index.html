このガイドについて

このガイドは、開発初心者がAIツールを活用して、効率的かつ安全にWebサイトの構築・修正ができるようになることを目指しています。

「いきなりコードを書かない」というのが、私たちの基本方針です。 まずChatGPTで要件を整理し、Cursorのプランモードで実装計画を立ててから、実装に入ります。 この流れを習得することで、ミスを減らし、品質の高い成果物を作れるようになります。

💭 なぜ「計画重視型」なのか？

Q：「計画を立てるより、直接コードを書いた方が早くない？」

A：確かに、小さな修正ならそうかもしれません。しかし、以下のような問題が起こりがちです：

  • 「あれ、何を修正すればいいんだっけ？」と迷う
  • 修正箇所を見落として、後で不具合が発覚
  • 他の人が見たときに「なぜこうしたのか」が分からない
  • 途中で別の案が出てきて、また最初からやり直し

計画を立ててから実装すると：

  • ✅ 何をすべきか明確になる
  • ✅ 見落としを防げる
  • ✅ チームでレビューしやすい
  • ✅ 後から振り返りやすい

対象読者

このガイドは、以下のような方を対象としています：

  • プログラミング経験が少ないが、Webサイトの作成・修正を担当する方
  • AIツール（ChatGPT、Cursor）を業務で活用したい方
  • 既存のWebサイトを安全に修正したい方
  • チームで開発する際の標準的なワークフローを学びたい方
前提知識： パソコンの基本操作（ファイル作成、フォルダ操作など）ができれば大丈夫です。 プログラミングの知識は不要です。

推奨される開発フロー

  1. 要件を整理する（ChatGPT活用）
    「何を作るのか」「何を修正するのか」をChatGPTと対話しながら明確にします。 漠然としたアイデアを、具体的な要件に落とし込みます。
  2. 実装計画を立てる（Cursorプランモード）
    Cursorのプランモードで、「どのファイルを」「どう修正するか」の計画を立てます。 AIが提案する計画をレビューし、必要に応じて調整します。
  3. プランを確認・承認する
    計画が妥当か確認します。必要に応じてチームメンバーにレビューを依頼します。
  4. 実装する
    承認された計画に基づいて、実装を進めます。 Cursorの内蔵ブラウザで随時確認しながら作業します。
  5. テスト・デプロイ
    動作を確認してから、本番環境にデプロイします。
ポイント： 「いきなり実装」ではなく、「要件定義 → プラン → 実装」の流れを守ることで、 ミスを減らし、品質の高い成果物を作れます。

💭 実際の作業イメージ

Q：「具体的にどんな感じで進めるの？」

A：例えば、「トップページに新しいセクションを追加したい」という場合：

  1. ChatGPTで要件整理：
    「トップページに『お客様の声』セクションを追加したい。3件の声を表示して、画像付きにしたい」 → ChatGPTが具体的な要件を整理してくれます
  2. Cursorプランモード：
    要件をCursorに貼り付けて、プランモードで計画を立ててもらいます → 「index.htmlにセクション追加、styles.cssにスタイル追加、images/フォルダに画像配置」といった計画が出てきます
  3. 計画をレビュー：
    「この計画で良さそう」と判断したら、実装開始
  4. 実装：
    プランに従って、Cursorが自動でコードを生成・修正してくれます
  5. 確認：
    内蔵ブラウザで表示を確認して、完成！

このように、「何をするか明確」→「計画あり」→「迷わず実装」という流れで進められます。

  • パソコン（Windows、Mac、Linuxのいずれか）
  • インターネット接続
