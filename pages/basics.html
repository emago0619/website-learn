1. GitとGitHubとは

Gitとは

Git（ギット）は、ファイルの変更履歴を管理するためのツールです。 例えば、Word文書を編集する際に「変更履歴」を残す機能がありますが、Gitはそれと似たようなものです。

Gitを使うことで、以下のことができます：

  • ファイルの変更履歴を保存する
  • 過去のバージョンに戻す
  • 複数の人で同じファイルを編集する
  • 変更内容を比較する

GitHubとは

GitHub（ギットハブ）は、Gitを使ったファイル管理をクラウド上で行うサービスです。 GoogleドライブやDropboxのように、ファイルをインターネット上に保存できますが、 コード（プログラム）に特化したサービスです。

GitHubを使うことで、以下のことができます：

  • コードをインターネット上に保存する
  • 他の人とコードを共有する
  • Webサイトを無料で公開する（GitHub Pages）
  • 変更履歴を視覚的に確認する

よく使うGitHubの用語

用語 説明
リポジトリ（Repository） プロジェクトのフォルダのようなもの。コードやファイルを保存する場所
コミット（Commit） 変更内容を保存すること。「保存」ボタンを押すようなもの
プッシュ（Push） ローカル（自分のパソコン）の変更をGitHubにアップロードすること
プル（Pull） GitHub上の最新の変更を自分のパソコンにダウンロードすること
ブランチ（Branch） メインのコードから分岐した別バージョン。実験的な変更に使う
クローン（Clone） GitHub上のリポジトリを自分のパソコンにコピーすること

2. AIエディタ（Cursor / Antigravity）とは

Cursorの特徴

Cursorは、AI機能が搭載されたコードエディタです。 通常のテキストエディタ（メモ帳など）と違い、コードを書くことに特化しており、 AIのサポートを受けながら効率的に作業できます。

Antigravityの特徴

Antigravityは、Google DeepMindが開発した「エージェント型」のAI IDEです。 Cursorの機能に加え、「自律型エージェント」が搭載されています。

  • 自律型エージェント
    「この機能を実装して」と頼むと、計画→実装→テストまでを自律的に行います。 人間は「承認」や「レビュー」に集中できます。
  • Gemini 3 Pro搭載
    Googleの最新AIモデルを搭載しており、複雑な推論や長時間のタスク実行が可能です。
  • ブラウザ操作
    エージェントがブラウザを操作して、実際の画面を確認しながら開発・修正を行います。

CursorとAntigravityの使い分け

特徴 Cursor Antigravity
得意なこと コードの補完、リファクタリング、チャットでの相談 タスクの丸投げ、自律的な実装、ブラウザテスト
人間の役割 AIと一緒に書く（ペアプログラミング） AIに指示して監督する（マネージャー）
おすすめの場面 細かい修正、既存コードの理解 新規機能の実装、大規模な変更、バグ調査

主な機能（共通）

  • AIアシスタント
    コードを書く際に、AIが提案や補完をしてくれます。 「この機能を作りたい」と伝えると、AIがコードを提案してくれます。
  • シンタックスハイライト
    コードを見やすく色分けして表示します。 エラーがある箇所も色で示してくれます。
  • ファイル管理
    プロジェクト内のファイルを整理して表示します。 フォルダ構造が一目でわかります。
  • Git統合
    GitHubとの連携が簡単で、変更履歴の確認やコミットがエディタ内でできます。

基本的な使い方

  1. ファイルを開く
    「File」メニューから「Open Folder」を選択し、プロジェクトのフォルダを開きます。
  2. AIに質問・依頼する
    • Cursor: Ctrl + K でコード生成、Ctrl + L でチャット
    • Antigravity: Task Boundary（タスク画面）でゴールを設定し、エージェントを起動
  3. 保存する
    Ctrl + S（MacはCmd + S）でファイルを保存します。
ヒント： どちらも無料で使用できます（制限あり）。 まずは両方触ってみて、自分に合う方を選ぶのがおすすめです。

3. CodexCLIとは

CodexCLIの役割

CodexCLIは、コマンドライン（ターミナル）からGitHubやGitの操作を行うためのツールです。 初心者の方には少し難しいかもしれませんが、このガイドに従えば問題なく使えます。

CodexCLIを使うことで、以下のことが簡単にできます：

  • GitHubリポジトリの作成
  • ファイルのアップロード
  • 変更のコミットとプッシュ
  • Webサイトの公開

コマンドラインとは

コマンドライン（ターミナル）は、マウスを使わずにキーボードだけで操作する画面です。 一見難しそうに見えますが、コマンドを覚えれば、とても効率的に作業できます。

CodexCLIの基本的なコマンド

よく使うコマンド一覧

コマンド 説明
codex init 新しいプロジェクトを初期化する
codex create repo GitHubに新しいリポジトリを作成する
codex push 変更をGitHubにアップロードする
codex status 現在の状態を確認する
codex --help ヘルプを表示する
注意： コマンドを入力する際は、大文字と小文字を正確に入力してください。 スペースも正確に入力する必要があります。

5. 静的なHPと動的なHPの違い

静的サイトとは

静的サイトは、HTML、CSS、JavaScriptだけで作られたシンプルなWebサイトです。 このガイドで作成するWebサイトも、静的サイトに該当します。

静的サイトの特徴：

  • ファイルがそのまま表示される（サーバーでの処理がない）
  • 動作が高速
  • セキュリティリスクが低い
  • GitHub Pagesなどで無料で公開できる
  • メンテナンスが簡単

動的サイトとは

動的サイトは、ユーザーの操作やデータベースの内容に応じて、 表示内容が変わるWebサイトです。

動的サイトの特徴：

  • サーバーサイドでプログラムが実行される（PHPやPythonなど）
  • ユーザーごとに異なる内容を表示できる
  • データベースと連携できる
  • ログイン機能やコメント機能が実装できる
  • より高度なサーバーが必要

どちらを選ぶべきか

静的サイトが向いている場合

  • 会社の紹介ページ
  • 個人のポートフォリオサイト
  • ブログ（更新頻度が低い場合）
  • ランディングページ
  • ドキュメントサイト

動的サイトが必要な場合

  • ECサイト（ショッピング機能）
  • SNS（ユーザー登録・ログイン機能）
  • 掲示板（コメント・投稿機能）
  • 会員制サイト
  • 検索機能が必要なサイト
初心者へのアドバイス： 最初は静的サイトから始めることをおすすめします。 静的サイトでも、十分に魅力的で機能的なWebサイトを作成できます。

6. PHPについて

PHPとは

PHP（ピーエイチピー）は、サーバーサイドで動くプログラミング言語です。 動的なWebサイトを作るときに、最もよく使われる言語の一つです。

PHPの役割

PHPは、以下のような処理を行います：

  • データベースからデータを取得して表示する
  • お問い合わせフォームのデータを処理する
  • ユーザーのログイン認証を行う
  • ファイルのアップロードを処理する
  • HTMLを動的に生成する

PHPの基本例

例えば、現在の日時を表示するPHPコード： 

<?php
echo "今日は " . date("Y年m月d日") . " です";
?>

このコードがサーバーで実行されると、実際の日付が表示されます。

WordPress とPHP

有名なブログシステム「WordPress」もPHPで作られています。 世界中のWebサイトの約40%がWordPressで作られていると言われています。

初心者へのアドバイス： 静的サイトを作る場合は、PHPの知識は不要です。 お問い合わせフォームやログイン機能など、より高度な機能が必要になったときに学び始めれば十分です。

7. Reactについて

Reactとは

React（リアクト）は、Facebookが開発したJavaScriptライブラリです。 モダンなWeb開発で非常に人気があります。

Reactの特徴

  • コンポーネントベース
    UI（ユーザーインターフェース）を部品として作成し、組み合わせて使う
  • 再利用可能
    一度作ったコンポーネントを、別のページでも使える
  • 高速な動作
    必要な部分だけを更新するので、動作が速い
  • 大規模開発に適している
    複雑なアプリケーションを効率的に開発できる

HTMLとReactの違い

通常のHTML

<div>
    <h1>ようこそ</h1>
    <p>これは静的なページです</p>
</div>

Reactの例

function Welcome() {
    return (
        <div>
            <h1>ようこそ</h1>
            <p>これはReactで作られたページです</p>
        </div>
    );
}

Reactが使われる場面

  • Facebook、Instagram、Netflix、Airbnbなどの大規模サービス
  • 複雑なUI操作が必要なアプリケーション
  • シングルページアプリケーション（SPA）
  • リアルタイムで更新されるダッシュボード
初心者へのアドバイス： Reactは強力ですが、学習コストが高いです。 まずは基本的なHTML/CSS/JavaScriptをマスターしてから、Reactに挑戦することをおすすめします。 シンプルなWebサイトなら、Reactは不要です。

8. Next.jsについて

Next.jsとは

Next.js（ネクストジェーエス）は、Reactをベースにしたフレームワークです。 Vercel社が開発しており、現代のWeb開発で最も人気のあるフレームワークの一つです。

Reactとの違い

ReactはJavaScriptライブラリですが、Next.jsはReactを使いやすくするための「フレームワーク」です。 車に例えると、Reactがエンジンなら、Next.jsは完成した車のイメージです。

Next.jsの主な特徴

  • サーバーサイドレンダリング（SSR）
    ページをサーバー側で生成してから送るので、SEOに強く、初期表示が速い
  • 静的サイト生成（SSG）
    ビルド時にHTMLを生成するので、超高速で表示できる
  • ファイルベースルーティング
    ファイルを作るだけで、自動的にページのURLが決まる
  • API Routes
    バックエンドのAPIも同じプロジェクト内で作れる
  • 画像最適化
    画像を自動で最適化してくれる

Next.jsが使われる場面

  • 企業のコーポレートサイト
  • ECサイト
  • ブログ・メディアサイト
  • ダッシュボード・管理画面
  • SEOが重要なWebアプリケーション

💭 会話形式で理解する

Q：「Next.jsって、普通のHTMLと何が違うの？」

A：大きく3つの違いがあります：

  1. 動的な内容を表示しやすい
    HTMLは固定の内容ですが、Next.jsならデータベースから最新情報を取得して表示できます
  2. ページ遷移が超高速
    ページ全体をリロードせずに、必要な部分だけを更新します
  3. 開発効率が高い
    コンポーネントを再利用できるので、同じコードを何度も書かなくて済みます

Q：「いつNext.jsを使うべき？」

A：以下のような場合にNext.jsを検討しましょう：

  • ✅ ページ数が多い（10ページ以上）
  • ✅ SEO（検索エンジン最適化）が重要
  • ✅ データベースと連携したい
  • ✅ 会員機能やログインが必要
  • ✅ 将来的に拡張する予定がある

逆に、シンプルな会社紹介ページなら、普通のHTMLで十分です。

使い分けのポイント： ランディングページや簡単な紹介ページは静的HTML、 本格的なWebアプリケーションやSEOが重要なサイトはNext.jsを検討しましょう。

9. Supabaseについて

Supabaseとは

Supabase（スーパーベース）は、「オープンソース版Firebase」と呼ばれる、 バックエンドサービス（BaaS: Backend as a Service）です。

簡単に言うと、「サーバーやデータベースを自分で用意しなくても、 会員登録やデータ保存などの機能が使えるサービス」です。

Supabaseの主な機能

  • データベース（PostgreSQL）
    データを保存・管理できます。SQLを書かなくても、管理画面から操作できます
  • 認証機能
    ユーザー登録、ログイン、パスワードリセットなどが簡単に実装できます
  • ストレージ
    画像やファイルをアップロードして保存できます
  • リアルタイム機能
    データが更新されたら、自動的に画面に反映されます（チャットアプリなどに便利）
  • API自動生成
    データベースのテーブルを作ると、自動的にAPIが生成されます

なぜSupabaseが人気なのか

  • 無料枠が充実している（個人プロジェクトなら無料で十分）
  • 設定が簡単（数分で使い始められる）
  • PostgreSQLを使っているので、後で移行しやすい
  • オープンソースなので、自前サーバーでも動かせる

Supabaseが使われる場面

  • 会員制サイト（ユーザー登録・ログイン機能）
  • ブログ・CMS（記事の保存・管理）
  • SNS・コミュニティサイト
  • タスク管理アプリ
  • 予約システム

💭 会話形式で理解する

Q：「Supabaseって何がすごいの？自分でサーバー立てるのと何が違う？」

A：大きな違いは「手間」です：

項目 自前サーバー Supabase
初期設定 数日〜数週間 数分
サーバー管理 自分で管理 不要
セキュリティ 自分で設定 自動で対応
スケーリング 手動 自動
費用 固定費 従量課金（無料枠あり）

つまり、Supabaseを使えば「開発に集中できる」というわけです。

Q：「Next.jsとSupabaseを組み合わせると何ができる？」

A：最強の組み合わせです！例えば：

  • ブログシステム
    Next.jsでフロントエンド、Supabaseで記事データを保存
  • 会員制サイト
    Supabaseの認証機能で会員管理、Next.jsでページ表示
  • リアルタイムダッシュボード
    データが更新されたら、自動的に画面に反映

この2つを組み合わせれば、本格的なWebアプリケーションが短期間で作れます。

活用例： 管理画面やWebアプリケーションを作る際に、Next.js + Supabaseの組み合わせは非常に強力です。 開発期間を大幅に短縮できます。

10. データベースが必要な場面

データベースとは

データベースは、大量のデータを整理して保存・管理するためのシステムです。 Excelの表のようなイメージですが、もっと高機能で大規模なデータを扱えます。

データベースの種類

  • MySQL / PostgreSQL
    最もよく使われる関係型データベース。WordPressなどで使用される
  • MongoDB
    NoSQLデータベース。柔軟なデータ構造に対応
  • SQLite
    小規模なアプリケーション向けの軽量データベース

データベースが必要な場面

データベースが必要な場合

  • ユーザー管理
    会員登録、ログイン機能がある場合
  • 投稿・コメント機能
    ブログやSNSのように、ユーザーが内容を投稿できる場合
  • ECサイト
    商品情報、注文履歴、在庫管理など
  • 検索機能
    大量のデータから条件に合うものを検索する場合
  • 予約システム
    予約状況を管理する場合

データベースが不要な場合

  • 会社の紹介ページ
    内容が固定で、更新頻度が低い
  • ポートフォリオサイト
    自分の作品を紹介するだけ
  • ランディングページ
    商品やサービスの紹介ページ
  • お知らせページ
    管理者が手動で更新する場合

静的サイトとデータベースの使い分け

機能 静的サイト データベース必要
ページの表示 ○ ○
お問い合わせフォーム △（外部サービス使用） ○
ログイン機能 × ○
コメント投稿 × ○
更新の手軽さ ○（ファイル編集） ○（管理画面から）
コスト 低（無料も可） 高（サーバー費用）
初心者へのアドバイス： 最初は静的サイトで十分です。 お問い合わせフォームは、Googleフォームなどの外部サービスを使えば、データベースなしでも実装できます。 本当にデータベースが必要になったときに、学び始めれば問題ありません。

9. 3つのツールの連携

GitHub、Cursor、CodexCLIは、それぞれ異なる役割を持っていますが、 組み合わせて使うことで、効率的にWebサイトを作成・管理できます。

基本的な作業の流れ

  1. CodexCLIでリポジトリを作成
    コマンドラインから、GitHubに新しいリポジトリを作成します。
  2. Cursorでファイルを編集
    CursorでHTMLやCSSファイルを作成・編集します。 AIのサポートを受けながら、コードを書きます。
  3. CodexCLIで変更を保存
    編集が完了したら、CodexCLIで変更をコミット・プッシュします。
  4. GitHubで確認・公開
    GitHub上で変更を確認し、必要に応じてWebサイトを公開します。
ヒント： 最初は難しく感じるかもしれませんが、 何度か繰り返すうちに、自然と手順を覚えられます。 焦らず、一つずつ進めていきましょう。
