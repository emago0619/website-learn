デプロイとは？

💭 デプロイって何？

デプロイとは、作成したWebサイトやアプリケーションをインターネット上で公開し、 誰でもアクセスできるようにする作業のことです。

簡単に言えば、「お店を開店する」ようなイメージです。 商品（Webサイト）を準備して、お店（サーバー）に並べて、お客さん（ユーザー）が来られるようにするのがデプロイです。

デプロイの基本的な流れ

  1. 開発環境でコードを書く
    自分のパソコン（ローカル環境）でWebサイトを作成します。
  2. テストする
    ローカルで動作を確認し、問題がないかチェックします。
  3. GitHubにプッシュ
    コードをGitHubにアップロードして、バージョン管理します。
  4. デプロイサービスと連携
    GitHub Pages、Vercel、Netlify、Cloudflareなどのサービスに接続します。
  5. 本番環境にデプロイ
    サービスが自動的にビルド＆デプロイを行い、URLが発行されます。
  6. 公開完了
    発行されたURLにアクセスすれば、世界中の誰でも見られるようになります！
💡 重要： GitHubにコードをプッシュするだけで自動的にデプロイされる仕組みを 「CI/CD（継続的インテグレーション/継続的デリバリー）」と言います。 この仕組みがあると、コードを修正して→プッシュするだけで→自動的にWebサイトが更新されます！

GitHub Pages 完全ガイド

GitHub Pagesは、GitHubが提供する無料の静的サイトホスティングサービスです。 初心者に最もおすすめのデプロイ方法です。

メリット

  • 完全無料 - 個人・商用利用ともに無料
  • 設定が簡単 - 数クリックで公開可能
  • 自動デプロイ - GitHubにプッシュするだけで自動更新
  • 独自ドメイン対応 - 自分のドメインも設定可能
  • HTTPS対応 - 無料で自動的にHTTPS化

デメリット

  • 静的サイトのみ（PHPやNode.jsサーバーは動かない）
  • パブリックリポジトリ推奨（プライベートでも可能だが制限あり）
  • 月間100GBの帯域制限

デプロイ手順

  1. リポジトリ作成
    GitHubで新しいリポジトリを作成し、コードをプッシュします。
  2. Settings → Pages
    リポジトリのSettingsタブから「Pages」を選択します。
  3. ブランチを選択
    「Source」で「Deploy from a branch」を選び、「main」ブランチと「/ (root)」を選択。
  4. 保存
    「Save」をクリックすると、数分で公開されます。
  5. URLを確認
    https://[ユーザー名].github.io/[リポジトリ名] にアクセス！
💡 Pro Tips:
  • リポジトリ名を [ユーザー名].github.io にすると、https://[ユーザー名].github.io というシンプルなURLになります
  • カスタムドメインは「Settings → Pages → Custom domain」で設定
  • GitHub Actionsと組み合わせることで、より高度なビルドプロセスも可能

Vercel 完全ガイド

Vercelは、Next.jsの開発元が提供する最高速のデプロイサービスです。 フロントエンドに特化した設計で、プレビューデプロイやエッジ機能が充実しています。

Vercelの特徴

  • ゼロコンフィギュレーション - 設定ファイル不要で自動検出
  • グローバルCDN - 世界中で高速にコンテンツ配信
  • プレビューデプロイ - Pull Request毎に自動でプレビュー環境生成
  • 環境変数管理 - 環境ごとに変数を簡単に管理
  • Vercel Functions - サーバーレス関数を簡単にデプロイ

デプロイ手順

  1. Vercelにサインアップ
    https://vercel.com にアクセスし、「Sign Up」をクリック。 GitHubアカウントでログインがおすすめ。
  2. Import Project
    ダッシュボードから「Add New... → Project」を選択。 GitHubリポジトリを選択します。
  3. 設定を確認
    Vercelが自動的にフレームワークを検出し、ビルド設定を提案します。 通常はそのままでOK。
  4. Deploy
    「Deploy」ボタンをクリック。数秒〜数十秒でデプロイ完了！
  5. URLを確認
    https://[プロジェクト名]-[ランダム文字列].vercel.app が発行されます。

環境変数の設定

Vercelでは、API キーなどの秘密情報を安全に管理できます。

  1. Settings → Environment Variables
    プロジェクトの「Settings」タブから「Environment Variables」を選択。
  2. 変数を追加
    「Key」に変数名（例：API_KEY）、「Value」に値を入力。
  3. 環境を選択
    • Production - 本番環境（mainブランチ）
    • Preview - プレビュー環境（Pull Request）
    • Development - ローカル開発環境
  4. 再デプロイ
    環境変数を追加したら、「Deployments」タブから最新のデプロイを「Redeploy」します。
💡 重要： Previewビルドでもアプリケーション内のNODE_ENVは"production"になります！ 環境を判別する場合はVERCEL_ENVシステム環境変数を使いましょう。

カスタムドメインの設定

  1. Settings → Domains
    プロジェクトの「Settings」→「Domains」を開きます。
  2. ドメインを追加
    「Add」をクリックし、自分のドメイン（例：example.com）を入力。
  3. DNS設定
    Vercelが指示するDNSレコード（AレコードまたはCNAMEレコード）を、 ドメイン管理サービスに設定します。
  4. 検証
    DNS設定後、数分〜数時間で自動的にHTTPS証明書が発行されます。

プレビューデプロイの活用

Vercelの最強機能の一つがプレビューデプロイです。

プレビューデプロイとは？

GitHubでPull Requestを作成すると、Vercelが自動的にそのブランチ専用のプレビュー環境を作成し、 一時的なURLを発行してくれます。

これにより、本番環境に影響を与えずに、チームメンバーやクライアントに 変更内容を確認してもらえます。

プレビューデプロイの流れ

  1. 新しいブランチを作成
    git checkout -b feature/new-feature
  2. コードを変更してプッシュ
    git add .
git commit -m "Add new feature"
git push origin feature/new-feature
  3. Pull Requestを作成
    GitHubでPull Requestを作成します。
  4. Vercel Botがコメント
    数秒後、VercelのBotがPull Requestにコメントを投稿し、プレビューURLを教えてくれます。
  5. プレビューを確認
    URLをクリックして、変更内容を確認します。
  6. マージ
    問題なければPull Requestをマージ。自動的に本番環境にデプロイされます。
💡 Pro Tips:
  • Vercel CLIをインストールすると、ローカルから直接デプロイできます：npm i -g vercel
  • 環境変数はローカルに取得可能：vercel env pull
  • ビルドログは「Deployments」タブで詳細に確認できます
  • 無料プランでも商用利用OK（個人プロジェクト）

Netlify 完全ガイド

Netlifyは、静的サイトホスティングに特化したサービスで、フォーム機能やサーバーレス関数が使いやすいのが特徴です。

Netlifyの特徴

  • 超高速デプロイ - 数秒でビルド＆デプロイ完了
  • Netlify Forms - HTMLフォームを自動でバックエンド化
  • Netlify Functions - AWS Lambdaベースのサーバーレス関数
  • Split Testing - A/Bテストが簡単
  • リダイレクト＆リライト - _redirectsファイルで柔軟な設定

デプロイ手順

  1. Netlifyにサインアップ
    https://www.netlify.com にアクセスし、 GitHubアカウントでログインします。
  2. Add new site → Import an existing project
    「Sites」タブから「Add new site」→「Import an existing project」を選択。
  3. GitHub連携
    「Deploy with GitHub」を選択し、リポジトリを選びます。
  4. ビルド設定
    ビルドコマンドと公開ディレクトリを設定します。
    • Next.js: ビルドコマンド npm run build、公開ディレクトリ .next
    • 静的HTML: ビルドコマンド不要、公開ディレクトリ / または dist
  5. Deploy site
    「Deploy site」をクリック。数秒〜数十秒で完了します。
  6. URLを確認
    https://[ランダム名].netlify.app が発行されます。

Netlify Formsの使い方

Netlifyの強力な機能の一つが、HTMLフォームを自動でバックエンド化してくれる「Netlify Forms」です。

  1. HTMLにフォームを追加
    <form name="contact" method="POST" data-netlify="true">
  <input type="hidden" name="form-name" value="contact" />
  <p>
    <label>名前: <input type="text" name="name" required /></label>
  </p>
  <p>
    <label>メール: <input type="email" name="email" required /></label>
  </p>
  <p>
    <label>メッセージ: <textarea name="message" required></textarea></label>
  </p>
  <button type="submit">送信</button>
</form>
  2. デプロイ
    GitHubにプッシュすると、Netlifyが自動的にフォームを検出します。
  3. 送信内容を確認
    Netlifyダッシュボードの「Forms」タブで、送信されたデータを確認できます。
💡 ポイント： data-netlify="true" を追加するだけで、自動的にバックエンド処理が追加されます。 サーバーを用意する必要がないので、超簡単です！

環境変数の設定

  1. Site settings → Environment variables
    サイトの「Site settings」→「Environment variables」を開きます。
  2. Add a variable
    「Add a variable」をクリックし、キーと値を入力します。
  3. 再デプロイ
    「Deploys」タブから「Trigger deploy」→「Deploy site」で再デプロイします。
💡 Pro Tips:
  • Netlify CLIで開発サーバーとFunctionsをローカルで動かせます：npm i -g netlify-cli
  • netlify.toml でビルド設定やリダイレクトルールを管理できます
  • 無料プランでは月100GBの帯域制限ですが、個人サイトなら十分

Cloudflare Pages 完全ガイド

Cloudflare Pagesは、2021年に正式リリースされた比較的新しいサービスです。 帯域無制限という圧倒的なメリットがあります。

Cloudflare Pagesの特徴

  • 完全無料 - 無料プランで十分な機能
  • 帯域無制限 - トラフィックが多いサイトでも安心
  • 月500回のビルド - Freeプランでも十分
  • Cloudflareの高速CDN - 世界中で高速配信
  • Workers統合 - エッジコンピューティングと簡単に統合
  • セキュリティ機能 - DDoS保護、WAFなど充実

デプロイ手順

  1. Cloudflareにサインアップ
    https://pages.cloudflare.com にアクセスし、 メールアドレスで新規登録します。
  2. Workers & Pages → Create application
    ダッシュボードから「Workers & Pages」→「Create application」→「Pages」タブを選択。
  3. Connect to Git
    「Connect to Git」をクリックし、GitHubを選択してリポジトリを選びます。
  4. ビルド設定
    フレームワークを選択すると、自動的にビルドコマンドが設定されます。
    • Next.js: npm run build
    • 静的サイト: 設定不要
  5. Save and Deploy
    「Save and Deploy」をクリック。数秒〜数十秒でデプロイ完了。
  6. URLを確認
    https://[プロジェクト名].pages.dev が発行されます。

環境変数の設定

  1. Settings → Environment variables
    プロジェクトの「Settings」→「Environment variables」を開きます。
  2. Add variable
    「Add variable」をクリックし、変数名と値を入力します。
  3. 環境を選択
    Production（本番）とPreview（プレビュー）を選択できます。
  4. 再デプロイ
    「Deployments」から最新のデプロイを「Retry deployment」で再実行します。

カスタムドメインの設定

  1. Custom domains → Set up a custom domain
    プロジェクトの「Custom domains」を開きます。
  2. ドメインを追加
    自分のドメインを入力します（例：example.com）。
  3. DNS設定
    Cloudflareが指示するDNSレコードを設定します。 Cloudflareでドメインを管理している場合は自動で設定されます。
  4. SSL証明書
    自動的にSSL証明書が発行され、HTTPS化されます。
💡 Pro Tips:
  • Cloudflare C3（npm create cloudflare@latest）を使うと、プロジェクト作成からデプロイまで一気にできます
  • Cloudflare Workersと連携することで、エッジで動的な処理も可能
  • 帯域無制限なので、画像や動画が多いサイトにも最適
  • DDoS保護など、セキュリティ機能が標準で付いてくる

💭 Cloudflare Pagesはどれくらい速い？

実際に使った人の感想では、「初めて触る人でも1時間あればDNS設定までできる」「5分でサイトを公開できた」 という声が多いです。

Cloudflareは世界中に300以上のデータセンターを持っているため、どこからアクセスしても高速です。

FTP / レンタルサーバー ガイド

FTP（File Transfer Protocol）は、ファイルをサーバーに直接アップロードする従来の方法です。 PHPやWordPress、データベースが必要な場合に使います。

いつFTPを使うべき？

  • PHPが必要なとき - 静的サイトホスティングではPHPは動かない
  • WordPressを使うとき - WordPress専用のレンタルサーバーが便利
  • データベースが必要なとき - MySQL、PostgreSQLなど
  • 完全なコントロールが欲しいとき - サーバー設定を自由にカスタマイズしたい

おすすめレンタルサーバー（日本）

サービス 料金 特徴
さくらのレンタルサーバ 月128円〜 老舗で安定、初心者向け
ロリポップ 月99円〜 低価格、WordPress簡単インストール
エックスサーバー 月990円〜 高速・高性能、人気No.1
ConoHa WING 月678円〜 国内最速、WordPress特化

FTPでデプロイする手順

  1. レンタルサーバーを契約
    上記のサービスから選び、契約します。
  2. FTP情報を確認
    契約後、FTPホスト名、ユーザー名、パスワードが送られてきます。
  3. FTPクライアントをダウンロード
    FileZilla（https://filezilla-project.org）や Cyberduckをダウンロードしてインストールします。
  4. FTP接続
    FTPクライアントを開き、ホスト名、ユーザー名、パスワードを入力して「接続」します。
  5. ファイルをアップロード
    左側（ローカル）から右側（サーバー）にファイルをドラッグ＆ドロップします。
  6. 確認
    ブラウザでサーバーのURLを開き、正しく表示されるか確認します。
⚠️ 注意点：
  • FTPは暗号化されていないため、SFTP（SSH File Transfer Protocol）を使うことを推奨
  • WordPressの場合、セキュリティ対策（アップデート、プラグイン管理など）が必要
  • 自動デプロイではないため、変更のたびに手動でアップロードする必要がある

トラブルシューティング

よくあるエラーと解決方法

❌ ビルドが失敗する

原因：

  • ビルドコマンドが間違っている
  • 依存関係がインストールされていない
  • Node.jsのバージョンが合わない

解決方法：

  1. ローカルで npm run build が成功するか確認
  2. package.json の engines でNode.jsバージョンを指定
  3. デプロイサービスのビルドログを詳細に確認

❌ デプロイは成功するが、ページが表示されない

原因：

  • 公開ディレクトリの設定が間違っている
  • ルーティングの設定が不足している（SPAの場合）

解決方法：

  1. ビルド後のファイルがどこに出力されるか確認（dist, build, .next など）
  2. SPA（Single Page Application）の場合、リダイレクト設定を追加
    • Netlify: _redirects ファイルに /* /index.html 200
    • Vercel: vercel.json に rewrites 設定

❌ 環境変数が反映されない

原因：

  • 環境変数を追加した後、再デプロイしていない
  • 変数名のプレフィックスが間違っている

解決方法：

  1. 環境変数追加後は必ず再デプロイする
  2. Vite: VITE_ プレフィックスが必要
  3. Next.js: NEXT_PUBLIC_ プレフィックス（クライアントサイドで使う場合）
  4. Create React App: REACT_APP_ プレフィックス

❌ カスタムドメインが動かない

原因：

  • DNS設定の反映待ち
  • DNSレコードの設定が間違っている

解決方法：

  1. DNS設定は反映まで数時間〜48時間かかる場合があります（待つ）
  2. dig コマンドや WhatsMyDNS でDNS伝播を確認
  3. AレコードまたはCNAMEレコードが正しく設定されているか確認

❌ デプロイが遅い

原因：

  • 依存関係が多すぎる
  • キャッシュが効いていない

解決方法：

  1. 不要な依存関係を削除
  2. node_modules のキャッシュ設定を確認
  3. 画像などの静的アセットはCDNを使う

さらに学ぶために

GitHub Actions で自動デプロイをカスタマイズ

GitHub Actionsを使うと、デプロイ前にテストを実行したり、複雑なビルドプロセスを自動化できます。

例：テストが通った場合のみデプロイする 

name: Deploy
on:
  push:
    branches: [main]

jobs:
  test-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm ci
      - run: npm test
      - run: npm run build
      - name: Deploy to Vercel
        run: vercel --prod --token=${{ secrets.VERCEL_TOKEN }}

パフォーマンス最適化

  • 画像最適化 - Next.js の <Image> コンポーネント、または sharp で圧縮
  • コード分割 - Dynamic Import でコードを分割し、初回読み込みを高速化
  • CDN活用 - 静的アセットはCDNから配信
  • キャッシュ戦略 - HTTP キャッシュヘッダーを適切に設定

セキュリティ対策

  • 環境変数を守る - .env ファイルは .gitignore に追加
  • HTTPS必須 - 現代のWebサイトはすべてHTTPS化が必須
  • CORS設定 - API を公開する場合は適切なCORS設定を
  • セキュリティヘッダー - CSP、X-Frame-Options などを設定

2025年現在、Webサイトをデプロイする方法は非常に多様化しています。 それぞれのサービスには特徴があり、プロジェクトに応じて最適なものを選びましょう。

初心者におすすめのデプロイサービス（まとめ）

  1. GitHub Pages - 完全無料、設定簡単、初心者に最適
  2. Vercel - Next.js/Reactに最強、プレビューデプロイが便利
  3. Cloudflare Pages - 帯域無制限、完全無料、高速
  4. Netlify - フォーム機能が便利、サーバーレス関数が使いやすい
🎉 次のステップ：

実際に手を動かして、自分のWebサイトをデプロイしてみましょう！ 最初は簡単なHTML+CSSのサイトから始めて、慣れてきたらNext.jsやReactを使った より高度なサイトに挑戦してみてください。
