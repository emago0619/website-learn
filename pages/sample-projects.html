プロジェクトの使い方

1. リポジトリをクローン

git clone https://github.com/username/project-name.git
cd project-name

2. 依存関係をインストール

# npm の場合
npm install

# yarn の場合
yarn install

3. 環境変数を設定（必要な場合）

# .env.example をコピー
cp .env.example .env.local

# .env.local を編集して必要な値を設定

4. 開発サーバーを起動

# 静的サイトの場合
# ブラウザでindex.htmlを開く

# Next.jsの場合
npm run dev

5. カスタマイズ

  • テキストを自分の内容に書き換える
  • 色やデザインをカスタマイズ
  • 不要な機能を削除、必要な機能を追加
  • 画像を差し替える

6. デプロイ

  • 静的サイト: GitHub Pages、Netlify、Vercel
  • Next.js: Vercel（推奨）、Netlify

カスタマイズのコツ

💡 段階的にカスタマイズしよう

  1. まず動かす： 最初は何も変更せず、そのまま動かしてみる
  2. テキストを変える： 見出しや説明文を自分の内容に
  3. 色を変える： CSS変数やTailwindの設定で色を調整
  4. 画像を差し替える： 自分の画像に入れ替える
  5. レイアウト調整： セクションの順番や構成を変更
  6. 機能追加： 新しい機能を少しずつ追加

⚠️ よくある失敗を避ける

  • 一度に大量に変更しない： 小さく変更して、その都度確認
  • 元のコードを残す： Gitでブランチを作って作業
  • README.mdを読む： プロジェクトの構造や設定方法が書いてある
  • エラーが出たら慌てない： エラーメッセージを読んで調査

🎨 デザインのカスタマイズ

CSS変数を使っているプロジェクトの場合：

:root {
  --primary-color: #6366f1;  /* メインカラー */
  --text-color: #1a1a1a;     /* テキスト色 */
  --bg-color: #ffffff;        /* 背景色 */
}

これらの値を変えるだけで、サイト全体の色が変わります。

サンプルプロジェクトを学び終えたら

🚀 次のステップ

  1. オリジナル機能を追加： サンプルにない機能を自分で実装してみる
  2. 複数のプロジェクトを組み合わせる： ブログ機能をポートフォリオに追加するなど
  3. 別の技術スタックで再実装： 同じ機能を違う技術で作ってみる
  4. 完全オリジナルのプロジェクトに挑戦： 0から自分でプロジェクトを企画・実装
