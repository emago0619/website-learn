目次

Google Stitchとは？

Google Stitchは、Googleが2025年5月のGoogle I/Oで発表したAI駆動のUI生成サービスです。 テキストや画像からWebサイトやアプリのデザインを自動生成し、HTML/CSSコードやFigmaファイルとして出力できます。

✨ Stitchの主な機能

  • テキストからUI生成：自然言語で説明するだけでデザインが完成
  • 画像からUI生成：手書きスケッチやスクリーンショットから生成
  • Figmaエクスポート：デザインをFigmaに直接貼り付け可能
  • コード生成：HTML/CSSコードを自動生成
  • レスポンシブ対応：スマホ・タブレット・PC対応デザイン
アクセス方法： https://stitch.withgoogle.com にGoogleアカウントでログイン（18歳以上、現在は無料）

なぜStitchを使うのか？

💭 従来の方法との違い

❌ 従来の方法（デザインなしで実装）

  1. いきなりコードを書き始める
  2. 「何となく」でレイアウトを決める
  3. デザインが統一されず、後で大幅修正
  4. レスポンシブ対応を後から追加して苦労

✅ Stitchを使った方法

  1. デザインを先に確定：実装前にビジュアルで確認
  2. 統一感のあるデザイン：AIが適切な配色・余白を提案
  3. レスポンシブ対応済み：最初から全デバイス対応
  4. 実装の叩き台：生成コードを元に効率的に実装
ポイント： Stitchは「デザイン確定 → 実装」という正しい順序を、初心者でも実現できるツールです。

まず、何を作るのかを明確にします。この段階でChatGPTを活用すると効果的です。

要件定義書テンプレート

# 要件定義書：レッスン一覧ページ

## 目的
ユーザーが利用可能なレッスンを見つけやすく、申し込みまでスムーズに行える

## 必要な要素
1. ヘッダー
   - サイトロゴ
   - ナビゲーション（ホーム、レッスン、マイページ、お問い合わせ）

2. フィルターバー
   - カテゴリー選択（プログラミング、デザイン、ビジネス、全て）
   - 価格帯フィルター

3. レッスンカード（グリッド表示）
   - サムネイル画像
   - カテゴリーラベル
   - レッスンタイトル
   - 短い説明文（2行程度）
   - 価格表示
   - 「詳細を見る」ボタン

4. フッター
   - コピーライト
   - リンク集

## デザイン要件
- カラーパレット：
  - プライマリー: #007bff (青)
  - セカンダリー: #6c757d (グレー)
  - 背景: #f8f9fa (ライトグレー)
  - テキスト: #212529 (ダークグレー)

- タイポグラフィ：
  - フォント: 'Noto Sans JP'
  - 見出し: 24px / 600
  - 本文: 16px / 400

- レイアウト：
  - デスクトップ: 3列グリッド
  - タブレット: 2列グリッド
  - スマホ: 1列スタック

- スペーシング：
  - カード間隔: 32px
  - カード内余白: 24px
  - セクション間: 48px

ステップ2-1: プロンプトの作成

ヒント： Stitchは英語での指示が最も精度が高いです。ChatGPTに日本語の要件定義を英語プロンプトに変換してもらうと便利です。

生成された英語プロンプト例

Create a modern lesson listing page for an online learning platform.

Layout and Structure:
- Header with logo and horizontal navigation (Home, Lessons, My Page, Contact)
- Category filter bar with chips/pills (Programming, Design, Business, All)
- Grid layout of lesson cards
- Footer with copyright and links

Lesson Cards (grid of 3 columns on desktop):
Each card should include:
- Thumbnail image at the top (16:9 ratio)
- Category label badge (top-right corner)
- Lesson title (h3, 20px)
- Short description (2 lines, 14px)
- Price tag
- "View Details" button

Design Style:
- Modern, clean, and minimal
- Card-based design with subtle shadows
- Rounded corners (8px)
- Smooth hover effects on cards

Color Palette:
- Primary: #007bff (blue)
- Secondary: #6c757d (gray)
- Background: #f8f9fa (light gray)
- Text: #212529 (dark gray)
- Card background: white

Typography:
- Font: 'Noto Sans JP' or similar sans-serif
- Headings: 24px, weight 600
- Body text: 16px, weight 400
- Line height: 1.6

Responsive Layout:
- Desktop (1200px+): 3-column grid
- Tablet (768px-1199px): 2-column grid
- Mobile (<768px): 1-column stack

Spacing:

- Card padding: 24px

Interactive Elements:

- Button hover state (darker blue)

💾 フェーズ3: デザインとコードの取得

方法1: Figmaにエクスポート（推奨）

  1. Stitchで「Export to Figma」ボタンをクリック
  2. Figmaが開き、デザインが自動的に配置される
  3. Figmaで細かい調整（色の微調整、フォント変更など）
  4. デザインのスクリーンショットを撮影（後でCursorに渡す用）
メリット： Figmaで調整することで、デザインを完璧に仕上げてから実装に移れます。

方法2: コードを直接取得

  1. Stitchで「Get Code」または「Export Code」をクリック
  2. HTML/CSSコードがコピーされる
  3. コードをテキストファイルに保存

ステップ4-5: 動的機能の追加

Stitchは静的なデザインを生成するため、インタラクティブな機能はJavaScriptで追加します。

// js/lessons.js

// レッスンデータ（実際はAPIから取得）
const lessons = [
  {
    id: 1,
    title: 'HTML基礎コース',
    description: 'Webページの基本構造を学びます',
    price: '無料',
    category: 'programming',
    image: 'images/html-course.jpg'
  },
  {
    id: 2,
    title: 'CSSデザイン入門',
    description: '美しいデザインを作る方法',
    price: '¥2,980',
    category: 'design',
    image: 'images/css-course.jpg'
  },
  // ... 他のレッスン
];

// ページ読み込み時に実行
document.addEventListener('DOMContentLoaded', () => {
  renderLessons(lessons);
  setupFilters();
});

// レッスンカードを生成
function renderLessons(lessonsToShow) {
  const grid = document.querySelector('.lesson-grid');
  grid.innerHTML = '';

  lessonsToShow.forEach(lesson => {
    const card = createLessonCard(lesson);
    grid.appendChild(card);
  });
}

// カード要素を作成
function createLessonCard(lesson) {
  const card = document.createElement('div');
  card.className = 'lesson-card';
  card.innerHTML = `
    <div class="card-image">
      <img src="${lesson.image}" alt="${lesson.title}">
      <span class="category-badge">${getCategoryLabel(lesson.category)}</span>
    </div>
    <div class="card-content">
      <h3>${lesson.title}</h3>
      <p>${lesson.description}</p>
      <div class="card-footer">
        <span class="price">${lesson.price}</span>
        <button class="btn-primary" onclick="viewLesson(${lesson.id})">
          詳細を見る
        </button>
      </div>
    </div>
  `;
  return card;
}

// フィルター機能
function setupFilters() {
  const filterButtons = document.querySelectorAll('.filter-chip');

  filterButtons.forEach(btn => {
    btn.addEventListener('click', (e) => {
      const category = e.target.dataset.category;

      // アクティブ状態を更新
      filterButtons.forEach(b => b.classList.remove('active'));
      e.target.classList.add('active');

      // フィルタリング
      if (category === 'all') {
        renderLessons(lessons);
      } else {
        const filtered = lessons.filter(l => l.category === category);
        renderLessons(filtered);
      }
    });
  });
}

// カテゴリーラベルを取得
function getCategoryLabel(category) {
  const labels = {
    programming: 'プログラミング',
    design: 'デザイン',
    business: 'ビジネス'
  };
  return labels[category] || category;
}

// レッスン詳細ページへ
function viewLesson(lessonId) {
  window.location.href = `lesson-detail.html?id=${lessonId}`;
}

💡 Tip 5: Stitchコードの扱い方

Stitchが生成するコードは「叩き台」として使います。

注意： Stitchのコードをそのまま本番に使うのは推奨されません。以下の理由があります：
  • 既存プロジェクトの構造に合わない
  • 最適化されていない（重複スタイルなど）
  • アクセシビリティが不十分な場合がある
  • SEO対策が含まれていない

推奨される使い方

  1. Stitchのコードから構造とスタイルのアイデアを得る
  2. 既存プロジェクトの規約に合わせて再実装
  3. Cursorに「Stitchのこのスタイルを参考に、既存のcss/styles.cssに統合して」と依頼

よくある問題と対処法

❌ 問題1: デザインが要件と違う

原因： プロンプトが曖昧、または情報不足

対処法：

  • より具体的なプロンプトに書き直す
  • 色コード、サイズ、余白を数値で指定
  • 参考画像を添付する

❌ 問題2: レスポンシブ対応が不十分

原因： プロンプトにレスポンシブ要件が不明確

対処法：

Please update the design to be fully responsive:
- Desktop (1200px+): [詳細]
- Tablet (768px-1199px): [詳細]
- Mobile (<768px): [詳細]

Ensure all interactive elements are touch-friendly on mobile (minimum 44px tap target).

❌ 問題3: Stitchのコードが既存プロジェクトで動かない

原因： クラス名の衝突、スタイルの優先順位

対処法：

  • Stitchのコードは参考程度に
  • Cursorに「既存プロジェクトの規約に合わせて再実装」を依頼
  • 既存のCSSクラスを再利用

❌ 問題4: Figmaエクスポートが上手くいかない

原因： ブラウザの権限、Figmaの設定

対処法：

  1. ポップアップブロッカーを無効化
  2. Figmaにログインしているか確認
  3. それでもダメなら「Get Code」でコードを取得

ベストプラクティスまとめ

✅ 推奨される手順

  1. 要件定義を最優先：ChatGPTで要件を明確化
  2. デザインを先に確定：Stitchでビジュアルを作成
  3. Figmaで仕上げ：細かい調整はFigmaで
  4. プランモードで計画：Cursorで実装計画を立てる
  5. 段階的に実装：コンポーネント単位で進める
  6. 随時確認：内蔵ブラウザでデザインと比較

✅ Stitchプロンプトのコツ

  • 英語で書く（精度が高い）
  • 具体的な数値を含める（色コード、px、余白）
  • 段階的に改善する（一度で完璧を求めない）
  • 参考画像を活用する
  • デザインシステムを定義する

✅ 実装のコツ

  • Stitchコードは「叩き台」として扱う
  • 既存プロジェクトの規約を優先
  • CursorにStitchデザインとコードを渡す
  • コンポーネント単位で実装
  • アクセシビリティとSEOを忘れない

まとめ

Google Stitchを使うことで、「デザイン → 実装」という正しい順序でWebページを作れるようになります。 特に開発初心者にとって、デザインを先に確定できることは大きなメリットです。

Stitchを使った開発の利点
  • ✅ デザインを実装前に確認・承認できる
  • ✅ 統一感のあるデザインをAIが提案
  • ✅ レスポンシブ対応が最初から含まれる
  • ✅ 実装の叩き台があるので迷わない
  • ✅ Figmaとの連携で調整も簡単

推奨ワークフロー：

  1. ChatGPTで要件整理
  2. Stitchでデザイン生成
  3. Figmaで微調整
  4. Cursorプランモードで実装計画
  5. 実装とテスト
  6. デプロイ

このワークフローをマスターすれば、品質の高いWebページを効率的に作成できるようになります。
