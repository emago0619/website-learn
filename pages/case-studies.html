ケース1: 会社のお知らせページに新機能を追加

📋 依頼内容

「お知らせページにカテゴリ絞り込み機能と検索機能を追加してほしい。今は全てのお知らせが表示されているだけで使いづらいという声が多い。」

1️⃣ ChatGPTで要件整理

タロウさんとChatGPTの対話

タロウ：お知らせページに絞り込み機能を追加したいんですが、どう進めればいいですか？

ChatGPT：まず現状を確認させてください。

  • お知らせのデータはどこから取得していますか？（HTML直書き？ JSON？ API？）
  • カテゴリはどんな種類がありますか？
  • お知らせは何件くらいありますか？

タロウ：HTMLに直接書いてます。カテゴリは「プレスリリース」「イベント」「採用」の3種類で、現在50件くらいあります。

ChatGPT：わかりました。では以下の要件でまとめましょう：

整理された要件（requirements.md）

# お知らせページ機能追加

## 目的
ユーザーが求める情報を素早く見つけられるようにする

## 機能要件
1. カテゴリ絞り込み
   - プレスリリース / イベント / 採用 / すべて
   - ボタンクリックで切り替え

2. キーワード検索
   - タイトルと本文を検索対象
   - リアルタイムで結果表示

3. 結果表示
   - 該当件数を表示
   - 該当なしの場合メッセージ表示

## 技術方針
- JavaScriptでクライアントサイド処理
- 既存デザインに統一
- レスポンシブ対応

2️⃣ Cursorプランモードで実装計画

Cursorへの指示：

requirements.mdの内容をもとに、お知らせページに絞り込み・検索機能を追加してください。
既存のnews.htmlを修正し、filter.jsを新規作成してください。

Cursorの提案内容

  1. news.html - カテゴリボタンと検索ボックスのUIを追加
  2. filter.js - 絞り込みと検索のロジックを実装
  3. styles.css - フィルター用のスタイルを追加

3️⃣ 実装

プランを承認して実装開始。5分ほどで完成。

4️⃣ 発生した問題と解決

問題1: 日本語検索がうまくいかない

原因： 大文字小文字を区別していた

解決： toLowerCase()を追加

問題2: 検索後にカテゴリ切り替えがリセットされる

原因： 状態管理が不十分

解決： 現在の絞り込み状態を変数で保持

5️⃣ テスト

  • ✅ 各カテゴリボタンで正しく絞り込まれるか
  • ✅ 検索ワードで正しく絞り込まれるか
  • ✅ カテゴリ+検索の組み合わせが動くか
  • ✅ スマホで操作できるか

📊 成果

  • 所要時間： 2時間（要件整理30分、実装30分、テスト・調整1時間）
  • Cursorクレジット： 約15回（プラン1回、細かい修正14回）
  • 学んだこと： 状態管理の重要性、日本語対応の注意点

ケース2: ECサイトの商品詳細ページをレスポンシブ対応

📋 依頼内容

「商品詳細ページがスマホで見づらい。画像が小さすぎたり、ボタンが押しにくかったりする。スマホ対応してほしい。」

1️⃣ 現状分析

ブラウザのDevToolsでスマホ表示を確認：

  • 画像が固定幅で小さい
  • テキストが画面からはみ出す
  • ボタンが小さくてタップしづらい
  • 横スクロールが発生

2️⃣ ChatGPTで要件整理

# レスポンシブ対応

## 修正箇所
1. 商品画像 - 幅100%、最大600px
2. 商品説明 - 適切な改行、フォントサイズ調整
3. 購入ボタン - タップしやすいサイズ（最小44x44px）
4. レイアウト - モバイルでは1カラム表示

## 対応デバイス
- スマートフォン（320px〜）
- タブレット（768px〜）
- PC（1024px〜）

3️⃣ Cursorでの実装

指示：

product-detail.htmlをレスポンシブ対応してください。
メディアクエリを使って、スマホ・タブレット・PCで最適な表示にしてください。

4️⃣ 実装内容

/* モバイル（デフォルト） */
.product-image {
  width: 100%;
  max-width: 600px;
}

.buy-button {
  width: 100%;
  min-height: 44px;
  font-size: 18px;
}

/* タブレット以上 */
@media (min-width: 768px) {
  .product-layout {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 40px;
  }
}

/* PC */
@media (min-width: 1024px) {
  .product-container {
    max-width: 1200px;
    margin: 0 auto;
  }
}

5️⃣ テスト

  • ✅ iPhone SE（小型）で表示確認
  • ✅ iPad（タブレット）で表示確認
  • ✅ PC（デスクトップ）で表示確認
  • ✅ 実機テスト（可能なら）

📊 成果

  • 所要時間： 1.5時間
  • 効果： スマホでの購入率が20%向上（仮想データ）
  • 学んだこと： モバイルファーストの重要性

ケース3: 問い合わせフォームにバリデーション追加

📋 依頼内容

「お問い合わせフォームで空白のまま送信されることが多い。必須項目のチェックと、メールアドレスの形式チェックを追加してほしい。」

1️⃣ 要件定義

  • 名前、メールアドレス、内容を必須に
  • メールアドレスの形式をチェック
  • エラー時は分かりやすいメッセージを表示
  • 送信成功時は完了ページに遷移

2️⃣ Cursorでの実装

HTML5のバリデーション + JavaScriptの組み合わせで実装：

<form id="contactForm">
  <input type="text" name="name" required>
  <span class="error" id="nameError"></span>

  <input type="email" name="email" required>
  <span class="error" id="emailError"></span>

  <textarea name="message" required></textarea>
  <span class="error" id="messageError"></span>

  <button type="submit">送信</button>
</form>

3️⃣ 発生した問題

問題: フォーム送信時に画面が遷移してしまう

原因： event.preventDefault()を忘れていた

解決：

form.addEventListener('submit', (e) => {
  e.preventDefault(); // これを追加
  // バリデーション処理
});

📊 成果

  • 所要時間： 1時間
  • 効果： 不完全な問い合わせが0件に
  • 学んだこと： フォームのイベント処理

全ケースから学ぶ共通のポイント

✅ 成功のポイント

  1. 要件を明確にする - ChatGPTで対話しながら整理
  2. 小さく始める - 一度に全部やろうとしない
  3. こまめにテスト - 変更のたびに動作確認
  4. 問題は記録する - 次回に活かせる
  5. コミットは頻繁に - いつでも戻れるようにする

⚠️ よくある失敗

  • 要件が曖昧なまま実装を始める
  • 一度に大量の変更をする
  • エラーメッセージを読まない
  • テストを後回しにする
  • バックアップを取らない

🎯 実践のコツ

1. 要件整理に時間をかける
実装時間の20-30%は要件整理に使うのが理想。急がば回れ。
2. プランモードを活用
実装前に計画を立てることで、見落としを防げる。
3. 困ったら休憩
煮詰まったら一度離れる。戻ってきたら解決策が見えることも。
